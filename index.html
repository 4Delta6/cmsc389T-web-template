Projects

Reverse Engineering Chat Server

I utilized tcp dump in conjunction with Wireshark to capture packets sent from a chat client to a chat server. By analyzing packets based on each chat command and different combinations of input, I was able to decode the commands header structure. The structure was typically 3 bytes of padding, 1 byte for the size of the message, 3 bytes to represent a constant/server identifier, then x bytes for the data payload which was typically split into 1 byte for argument length then the data. With this structure, I was able to recreate the chat servers full functionality in Rust.

In Rust, I was able to utilize ARCs, Mutexes, and Threads to handle multiple clients at once to provide the functionality of direct messaging, joining/leaving chat rooms, listing out the users in a chat room/entire server, and changing your nickname.
